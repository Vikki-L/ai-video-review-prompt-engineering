# 01 · Problem & Context

## 🌍 Business Context

某北美 AI 教育出海产品的获客模式：**雇佣 600+ 北美校园大使**在 TikTok / Instagram / YouTube 拍短视频引流。这些视频由 **Coach 团队**（10+ 外部 Coach + 内部 POC）审核，每周处理 **2000+ 条**。

## 🎯 The Concrete Problem

人工视频审核环节痛点反复出现：

| 痛点 | 后果 |
|------|------|
| **耗时长** | 一位 Coach 一天审核几十条已经满载，挤压指导和策略时间 |
| **不同 Coach 标准不一致** | 同一条视频在不同 Coach 那里可能拿到完全不同的反馈 |
| **新 Coach 上手周期长** | 没有"参考答案"，新 Coach 要花数周观察才能跟上团队节奏 |
| **错误案例（badcase）无沉淀** | 同样的低质视频问题反复出现，团队学不到 |

## 🔍 What I Set Out To Build

不是"做一个 AI 自动审核小工具"，而是一个**能直接嵌入 Coach 日常工作流的多步审核 pipeline**：

1. **多步链路**：业务类别识别 → 模板匹配 → AI 多步审核 → 人工复核
2. **可解释**：AI 不是给个分数，而是按维度拆解给出 reason
3. **可信任**：通过反幻觉策略让输出稳定可消费
4. **能进化**：badcase 反哺机制让 AI 越用越准

## 🧠 Key Design Decisions

| 决策 | 理由 |
|------|------|
| **多步审核而非单次评分** | 一次 prompt 让 AI 同时判断"类型 + 各维度 + 决策" → AI 容易混乱 |
| **按视频类型分类的 Prompt 体系** | 不同类型视频的评判维度完全不同（一个 prompt 包打不了）|
| **客观硬否决规则前置** | 把"主观判断"前置成"客观过滤"，降低 AI 主观波动 |
| **JSON Schema enum 锁死输出** | 杜绝 AI 输出"basically OK"这种模糊词 |
| **decision 字段** | 主观"评级"→ 客观决策，便于下游自动消费 |
| **Badcase 闭环** | AI 不能学，只能由人喂 → 设计反哺机制让错误案例进入下次 prompt |
